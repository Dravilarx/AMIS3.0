Módulo de Agendamiento de Procedimientos Médicos Intervencionales (AMIS 3.0)

La concepción central de este módulo es establecer un sistema robusto y eficiente para la gestión integral del agendamiento de diversos procedimientos médicos. Este sistema debe permitir la asignación de procedimientos a múltiples médicos especialistas y su programación en diferentes centros clínicos asociados. El objetivo primordial es optimizar la coordinación y planificación de la atención médica intervencional.

1. Repositorio de Procedimientos y Precios:
El sistema debe incorporar un exhaustivo repositorio centralizado de todos los procedimientos que pueden ser agendados. Cada procedimiento debe estar completamente caracterizado y codificado. Además, este repositorio debe incluir de manera obligatoria la información de los costos o precios asociados a cada procedimiento, lo cual es fundamental para la transparencia y la gestión financiera.

2. Gestión de Requisitos Médicos y Baterías de Requisitos:
Se requiere un repositorio dinámico de requisitos médicos. Estos requisitos son, en su mayoría, documentos necesarios (por ejemplo, consentimientos informados, exámenes previos, autorizaciones). La funcionalidad clave es que, al seleccionar un conjunto de estos requisitos, se configure una "batería de requisitos" específica. Estas baterías deben poder ser asignadas de forma flexible, ya sea según el tipo de procedimiento médico que se agenda o según el centro clínico donde se realizará.
Adjunto de Documentos: El sistema debe contar con la funcionalidad para que el usuario (o el personal administrativo) pueda adjuntar los documentos correspondientes que cumplen con los requisitos definidos.
3. Control y Verificación de Requisitos (Check-Out):
Es imperativo implementar un sistema de check-out o verificación de requisitos. Este mecanismo permitirá llevar un control riguroso de cuáles requisitos han sido presentados y validados. Solo una vez que todos los requisitos obligatorios para un procedimiento específico hayan sido marcados como completados, se podrá avanzar a la fase final de planificación y confirmación definitiva del procedimiento agendado.

4. Repositorio y Envío de Indicaciones:
Se necesita un repositorio de indicaciones médicas predefinidas. Estas indicaciones son cruciales para la preparación del paciente y deben ser asignadas de acuerdo al procedimiento específico y al lugar donde se llevará a cabo.
Formatos de Comunicación: Para asegurar la correcta entrega de información al paciente, el sistema debe generar las indicaciones en dos formatos distintos:
Formato Extenso para Correo Electrónico: Un formato detallado y estructurado, apto para ser enviado por correo electrónico.
Formato Corto para WhatsApp: Un formato conciso y de fácil lectura, optimizado para su envío a través de la aplicación de mensajería instantánea WhatsApp.
5. Visualización y Accesibilidad del Agendamiento:
La interfaz del sistema debe ser diseñada para una visualización fácil e intuitiva del agendamiento. Se deben ofrecer múltiples vistas:
Vistas de Tablas: Para la gestión detallada y la capacidad de filtrar y ordenar datos.
Vistas de Calendarios: Para una perspectiva temporal clara de los procedimientos programados.
Resúmenes de Agendas: Para que los médicos puedan acceder rápidamente a sus cronogramas.
Opciones de Exportación e Impresión: Se debe permitir la exportación de las agendas y resúmenes (por ejemplo, a formatos como Excel o PDF) para su visualización fuera del sistema, además de la funcionalidad de impresión directa de las agendas.
6. Integración con el Módulo de Logística:
El módulo AMIS 3.0 debe establecer una relación directa y funcional con un módulo de Logística. Esta integración es fundamental para la coordinación eficiente de recursos cuando los procedimientos se realizan en ciudades o ubicaciones geográficas distintas a la base. La relación debe permitir:
Coordinación de gastos de traslados.
Gestión y control de viáticos.
Administración de otros gastos operativos relacionados con la ejecución del procedimiento fuera de la sede principal.
7. Alcance de los Procedimientos Médicos:
El sistema está enfocado en la gestión de procedimientos relacionados con la Radiología Intervencional, incluyendo, pero no limitándose a:
Biopsias (guiadas por imágenes).
Punciones diagnósticas o terapéuticas.
Drenajes (percutáneos).
8. Datos Esenciales para el Agendamiento del Paciente:
La ficha de agendamiento debe capturar la información demográfica y clínica básica necesaria para la identificación y contacto del paciente, así como la gestión administrativa:
Nombre Completo del Paciente
RUT (Rol Único Tributario o documento de identificación equivalente)
Correo Electrónico
Número de Teléfono
Previsión de Salud (Isapre, Fonasa, u otro seguro/convenio)
Fecha de Nacimiento
Dirección de Residencia
Institución o Entidad Referente (si aplica)
9. Funcionalidades de Búsqueda Avanzada:
La herramienta de búsqueda debe ser diseñada para ser extremadamente potente y rápida. Debe permitir filtrar o buscar un paciente de manera eficiente utilizando los siguientes criterios:
Nombres y Apellidos del Paciente
RUT del Paciente
Médico Tratante o Intervencionista
10. Gestión de Resultados (Post-Procedimiento):
Dado que muchos procedimientos finalizados, como las biopsias, generan resultados clínicos (informes anatomopatológicos, etc.), el sistema debe contemplar una funcionalidad para la gestión de estos documentos. Los resultados de los procedimientos deben poder ser adjuntados al registro del paciente para una referencia completa y accesible.
